# 서비스 상태 패널 최소 설계

## 목표

운영자가 한눈에 볼 수 있도록 4개 서비스 상태를 표시한다.

- **api-server** — API 서버 가동 여부
- **discord-operator** — Discord 봇 가동 여부
- **market-bot** — 마켓 봇(엔진 호스트) 응답 여부
- **engine** — 매매 엔진 가동 여부 (RUNNING / STOPPED)

## 구현 위치

| 구분 | 위치 | 설명 |
|------|------|------|
| API | `GET /api/services-status` | `api-server`에서 제공. `apiServer`, `marketBot`, `engineRunning` 반환. |
| 표시 | Discord **현재 상태** 임베드 | 패널 버튼 "📡 현재 상태" 또는 슬래시 `/status` 응답 상단에 "서비스 상태" 필드로 표시. |

## 응답 형식 (GET /api/services-status)

```json
{
  "apiServer": true,
  "marketBot": true,
  "engineRunning": false
}
```

- **apiServer**: 이 API가 응답하면 `true` (api-server 자체가 살아 있음).
- **marketBot**: market-bot(`MARKET_BOT_URL`) `/status` 호출 성공 여부.
- **engineRunning**: market-bot `/engine-status`의 `status === 'RUNNING'` 여부.

discord-operator 상태는 api-server가 직접 알 수 없으므로, **Discord에서 "현재 상태"를 호출한 경우**에만 "discord-op 🟢"로 표시한다 (응답 주체가 Discord 봇이므로 가동 중으로 간주).

## Discord 임베드 표시 예

**서비스 상태**  
`api-server 🟢 · discord-op 🟢 · market-bot 🟢 · engine 🔴`

- 🟢: 정상/가동
- 🔴: 비가동 또는 미응답

## 확장 시 고려사항

- **discord-operator 헬스체크**: 나중에 discord-operator가 api-server에 주기적으로 heartbeat를 보내면 `GET /api/services-status`에 `discordOperator: true/false`를 추가할 수 있음.
- **select menu / modal**: interaction handler에 `isStringSelectMenu()`, `isModalSubmit()` 분기를 추가하면 동일한 서비스 상태 필드를 다른 UI에서도 재사용 가능.
